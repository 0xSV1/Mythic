Welcome to Apfell, where nothing is a bug and everything is a feature

Apfell is meant to be a post-exploit, collaborative, red teaming framework.
Check out my GitHub website or my GitHub Repos for more information on the process of creating Apfell, code repos, and blog posts on updates.

Be sure to go to "Manage Operations"->"Operations Management" and make an operation your "current operation"


What is this Apfell framework?

Apfell is meant to be an extendable user interface that allows operators to create/adjust/control agents in an easier fashion than starting from scratch.
For information on agent specific commands, check out: API -> Payload Type Command Help
For information on the available RESTful endpoints, check out: API -> RESTful API
For information on how to create a new agent or leverage some internal features, check out: API -> C2 Documentation

How can I quickly start using this framework?

  1. Select an operation to be your current operation (this affects what payloads, callbacks, and tasking you see)
    • Manage Operations -> Operations Management
    • Select an operation to "Make Current". You'll see this reflected in the top navigation bar
  2. If you want to use a C2 profile other than the default one, you need to start it
    • Manage Operations -> C2 Profiles Management
    • Potentially update the configuration for the profile via "Edit Files", download, modify, and re-upload
    • Click "Start"
  3. Create your payload
    • Create Components -> Create Base Payload
    • Select your C2 Profile
    • Fill in any required parameters (some may be auto-populated with hints or default values)
    • Select a payload type that supports this profile
    • Supply a name for your payload
    • Supply a tag for your payload. This tag will auto-populate for the description of any callbacks based on this payload.
    • Select any/all commands you want to be initially available in this payload. Depending on the payload type, you can load more commands after you get a callback.
    • Click "Submit". This will either submit the tasking to the corresponding container or, if it's an external payload type, then it just registers the payload and data in the database.
  4. View/Download/host the payload
    • Manage Operations -> Payload Management
      • Click "Download" to download the payload to your system
      • Click "Config" to view the metadata about your payload
      • Click "UUID" to view the unique identifier for the payload
    • To host the payload for execution:
      • Services -> Host Files
      • Click "Host New File"
      • You can either upload a file to host or, since the payload is already on the Apfell server, check "Local File"
        • Paste in the payload's UUID and the system will automatically pull all the info for it
  5. Execute your payload
  6. You'll get a new callback in Operational Views -> Active Callbacks
  7. Click "Interact" on a callback and start typing commands to use your new agent
